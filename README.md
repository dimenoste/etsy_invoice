# Etsy → Romanian Invoice Generation System

---

## Overview

This system generates a Romanian-style invoice (**Factura**) for each Etsy order by combining:

- financial transaction data from Etsy CSV exports
- customer identity and address data extracted from Etsy PDF invoices

The generated invoice contains:

- seller (issuer) information from configuration
- buyer identity and address from Etsy PDF invoice
- order date from CSV transaction data
- one simplified item line per order
- financial summary:
  - revenue
  - fees
  - taxes
  - net result

---

# Current Financial Computation Logic

For each Etsy order ID, the system reconstructs financial information using CSV transaction rows.

---

## 1. Revenue

Revenue is computed as:

```text
SUM(all "Sale" rows)
```

This represents the gross income generated by the order.

---

## 2. Platform Fees

Currently included platform costs:

- transaction fees
- payment processing fees

These are deducted from revenue.

---

## 3. Taxes

Sales tax rows from the CSV are handled separately from revenue.

They are included as tax-related deductions.

---

## 4. Net Result

Current formula:

```text
net = revenue
    − transaction fees
    − processing fees
    − sales tax
```

---

# Important Missing Fiscal Components

The current implementation is **not fully fiscally complete**.

Two important Etsy expense categories are still missing from the computation pipeline.

---

## 1. Listing Fees — Missing

Etsy listing fee rows exist in the CSV export but are not consistently included in fee aggregation.

### Consequence

- platform expenses are underestimated
- net income is overstated

---

## 2. Advertising / Marketing Costs — Missing

Etsy Ads and other marketing-related charges are present in the CSV data but are not integrated into the financial summary.

### Consequence

- operational expenses are incomplete
- profitability calculations are inaccurate

---

# Accounting Impact

Because listing fees and advertising expenses are excluded:

- total expenses are incomplete
- computed profit is artificially higher than reality
- generated invoices do not fully reflect Etsy's complete fee structure

---

# Correct Interpretation of the Current System

The current system produces:

> a partially complete fiscal reconstruction of Etsy orders

---

## Currently Handled Correctly

The system correctly processes:

- revenue
- transaction fees
- payment processing fees
- sales tax
- customer identity extraction
- invoice generation structure

---

## Still Missing

The system still does **not** properly include:

- listing fees

- accounting-grade accuracy

- complete Etsy fiscal reconstruction
